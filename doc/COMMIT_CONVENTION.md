# Commit Convention

## 커밋 메시지 형식

```
<<type>> <subject>

<body>

<footer>
```

---

## Type 종류

| Type       | 설명                              |
|------------|-----------------------------------|
| `feat`     | 새로운 기능 추가                  |
| `fix`      | 버그 수정                         |
| `docs`     | 문서 수정                         |
| `style`    | 코드 포맷팅, 세미콜론 누락 등 (기능 변경 없음) |
| `refactor` | 코드 리팩토링                     |
| `test`     | 테스트 코드 추가 및 수정          |
| `chore`    | 빌드, 패키지 설정 등 기타 변경    |

---

## 규칙

- 헤더는 `<type>` 형식으로 꺾쇠(`< >`)를 사용 (예: `<fix> 긴급Fix`)
- 커밋 메시지(subject, body)는 **한글**로 작성
- `subject`는 50자 이내로 작성
- 마침표(`.`) 사용 금지
- 현재 시제 사용 ("추가한다" O, "추가했다" X)
- `body`는 변경 이유와 내용을 72자 이내로 작성
- `footer`는 관련 이슈 번호 기재 (예: `Closes #123`)

---

## 예시

```
<feat> 로그인 기능 추가

이메일과 비밀번호를 이용한 로그인 기능을 구현한다.
JWT 토큰 기반 인증 방식을 적용한다.

Closes #42
```

```
<fix> 긴급Fix
```
